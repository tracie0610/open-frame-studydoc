Spring 和Struts 中BeanUtils的一点区别和hibernate 遇到的一个异常

spring:BeanUtils.copyProperties(a(赋予b),b(获得a)）；

Struts中相反


ids for this class must be manually assigned be for e calling save ()

hibe rnate出现这个错误的原因有可能因为，你的表中有个主键。 但是你插入的（调用save )时那个值是null

在设置成assigned 时报：ids for this class must be manually assigned be for e calling save ()：
在设置成native,sequence时同报：org.hibe rnate.exception.GenericJDBCException： could not get next sequence value


解决办法:
如果是assigned ，就必须给id赋值，如果是native、increment就不用了；

如果用assigned 的话程序里一定要自行设定主键，如果你数据库表id字段使用autoincrement话可以用native
