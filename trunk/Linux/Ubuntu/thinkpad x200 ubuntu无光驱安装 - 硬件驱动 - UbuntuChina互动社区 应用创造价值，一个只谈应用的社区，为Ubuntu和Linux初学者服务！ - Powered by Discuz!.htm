thinkpad x200 ubuntu无光驱安装

http://hi.baidu.com/dillisbest/b ... 4de486810a1883.html

日志原文：http://forrestrun.blog.sohu.com/106666740.html

thinkpad x200没有光驱，买了一个假光驱引导非常不好使，因此考虑用硬盘版安装一个ubuntu。

使用的是8.10-desktop.iso，放在e盘（fat32），并拷贝出来.disk目录、casper\vmlinuz, casper\initrd.gz，

然后在grub4dos的menu.lst文件中增加一个title。

title Ubuntu 8.10 LiveCD
root (hd0,5)
kernel /vmlinuz boot=casper iso-scan/filename=/Ubuntu-8.10-desktop-i386.iso ro quiet splash locale=zh_CN.UTF-8
initrd /initrd.gz
boot

我的e盘是sda6对应的是（hd0,5）.

重启用安装的dos引导,进入e盘,

e:\grub.exe

不过出现如下提示... ...

Failure restore ROM INI 0x60C vector.Unsupported DOS,device driver,or TSR

搜了一下,有人说要根据机型去调试... 用到debug查看中断....或者是什么纯dos,之类的...

怎么那么麻烦的...???

尝试了一下grub4dos里的其他命令

e:\bootlace.com -l 0x80(似乎是这个命令...)

之后再次重启,似乎多了一个加在步骤,会去寻找grub里的grldr...

于是呼.将grub4dos解压到e分区,重启...(虽然有很多的文件,但是不知道到底需要什么,所以就全部啦,..)

重启之后.引导... 出现了...安装界面(和平常将光盘放进去是一样的)

启动后在分区的地方无法识别分区表，

打开终端,$df(就可以看到有一个挂载项是isodevice)

原因和硬盘模拟过程有关，

处理方法:进入shell， 看.iso文件所在的分区是否被虚拟到iso了，

然后强制umount这个分区，sudo umount -l /isodevice

然后后退一步,再重新分区就行了。

http://digitalsonic.javaeye.com/blog/262788
