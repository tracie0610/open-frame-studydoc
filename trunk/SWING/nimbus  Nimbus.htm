Nimbus - A New Look-and-Feel for Swing

Nimbus is the name of a look-and-feel designed by Sun for the Java Desktop System; it's implemented as a GTK theme in the latest Solaris 11 pre-release builds. In 2007, this project was started by Sun's Swing Team and Ben Galbraith to build a pure Java implementation of this look and feel under a opensource licence. Later on it was decided to pull Nimbus LAF in to the core of the Java platform as part of version 6u10. Nimbus is now available in the beta version of 6u10 available at http://download.java.net/jdk6/.

The version in the JDK took some of the concepts that were experimented with in the project but is a complete new codebase. It is still based on Synth LAF but is 100% pure vector based using no images. This project is dead for the time being until Nimbus is made opensource through Open JDK. Once that happens it might be pulled back into the project so that we can work on a backport jar that could run on older Java versions than 6u10.

Old Info:

Nimbus is the name of a look-and-feel designed by Sun for the Java Desktop System; it's implemented as a GTK theme in the latest Solaris 11 pre-release builds. In 2007, Sun's Swing Team and Ben Galbraith jointly launched an open-source project to implement the Nimbus specifications as a Swing look-and-feel. Jasper Potts, an experienced Swing developer with notable experience created high-quality Swing look-and-feels, was recently hired by Sun and provided the initial code for the project, including a baseline Nimbus Synth implementation of a large number of components.

Currently, the project is using Synth to implement Nimbus. This dovetails nicely with the Swing Team's goal to further refine Synth, but the primary reason for this decision is that using Synth does make many tasks easier. While Java 5 compatibility is a goal, this project will first implement Nimbus on Java 6 and will address Java 5 later, either through a Java 5 update or a stand-alone compatibility library.

There is a webstartable demo of SwingSet running Nimbus on Swing Labs.

Instructions

  • SwingLabs The nimbus project is part of SwingLabs and there will be nightly builds etc available on swinglabs.org.
  • Large Repository. The Subversion repository for this project is quite large (~340 MB) because much of the artwork has been checked in. To make it easy to avoid checking out the artwork, we've organized the repository in this way: 
        [root]/
            code/
            docs/
            www/
    You can avoid checking out the artwork by using [root]/code/ as your repository URI (i.e., https://nimbus.dev.java.net/svn/nimbus/trunk/code)
  • Help Out! We can use all the help we can get; there's plenty to do. Once you've been added to the project as a developer, join the developer mailing list and check out the project plan (or bug list) to see what needs doing. As always, best to communicate with other developers on the list before tackling something to ensure your efforts don't go wasted.
