3.2  使用<html:img/>显示从一个属性文件映射的图像

Rendering images mapped from a properties file using <html:img/>

◆ 问题

怎样使用<html:img/>标签来得到在一个属性文件中映射的图像引用。

◆ 背景

<html:img/>是通过servlet 引擎处理成为一个HTML <img/>标签的。这个标签的图像路径是依靠创建它时设置的属性来解析的。你必须 使用以下四个属性之一：src、page、action或者srcKey.src。最通用的属性src 把URL 重写应用于你指定的图像的具体的路径。Page 提供被显示图像的相对于模块的路径的URL。action附加一个显示请求的图像的命名Action，以一个(/)开始。最后，srcKey.src 引用在一个以 bundle 属性命名的资源包中的图像的URL（如果没有指定包，使用默认包）。这些是最通用的属性，通过无所不在的HTML <img/>标签实现，并没有包括全部的属性。

n         align ―― 图像的定位。

n         alt ―― 在图像显示之前用来显示的文本，或者如果图像不能被显示的时候所显示的文本。

n         height ―― 图像的高度。

n         width ―― 图像的宽度。

n         border ―― 以像素为单位来显示的图像边界。

n         src ―― 被显示的图像的位置。

Struts <html:img/>标签不仅支持和实现以上全部的图像属性（正如它的HTML 对应物一样），它也支持Struts 的连接和URL 操作机制。在这个方法中，我们将展示怎样使用标签的srcKey 属性来指向一个Struts 消息资源属性文件中的URL 引用，在单一的属性文件中，有效地存储所有图像引用。当应用程序中有许多图像的时候，或者有特别的为了在远程服务器上高速缓存目的的URL 链接的时候，这特别有帮助，例如Akamai等。一旦图像引用被存储在一个消息资源属性文件之内，出现重复入口的可能性被显著地减少，这使管理图像和它们的相关目录的时间大大减少。

◆ 方法

把图像合并到web 应用程序中是必要的。它们增加风味，给予风格，并且扩展有关页面的打印信息。外观对于用户体验是重要的，但是在应用程序被部署的前后，经营管理这些图像对于开发者来说更加重要。

图像位置和检索是开发者的重要的考虑，但是其他的考虑也会有。这些图像本地化了吗？法国人需要一个不同的图像吗?它们的链接怎么样？Struts <html:img/>标签通过它

的丰富的属性集极好地回答了这些问题。通过在这个标签里设置一些属性，我们将展示如何实现几年前需要许多代码行才能实现的功能。

最优方法

在大多数情况下保持单一的图像目录。――单一图像目录应该用于大多数情况。这个目录最好位于一个于页面目录相同的级别。用无关的、嵌套的、或者多重目录来处理图像，随着应用程序的增长，这会是非常麻烦和特别不好理解的。记得这些图像最后不只是将要从页面（例如，Tile、布局模板、脚本、和CSS 目录）被访问是很重要的。

让我们看一个在Struts JSP 页面上的典型图像，解剖它，并且通过在Struts 框架之内使用属性文件来提高对页面中加入图像的理解。下面代码片断展示Struts-config.xml 文件中的一个入口，它指定一个图像资源包（对于资源包的更多信息，见方法1.5 “在配置中指定一个资源属性文件”）。

<message-resources parameter="resources.images" key="images"/>

在清单3.2中的JSP页面包含一个介绍消息和一个图像引用，指向以上所述资源包。

清单3.2  使用Struts <html:img/> 标签的JSP 页面

<html:html locale="true">

<head>

<html:base/>

<title><bean:message key="imagePage.title"/></title>

</head>

<body>

   <bean:message key="imagePage.IntroductionMessage"/><br/>

<html:img pageKey="images.foo.reallyBigImageWithLink.jpg"   b

           bundle="images" height="175" width="300" styleClass="bigimages"/>        c

</body>

</html:html>

我们已经涉及使用pageKey 属性（b处）的图像；随后引用一个在image 资源包中与它的具体的键-值对相关的引用。该图像引用包含适当的页面上下文――<html:img/>标签在被servlet 引擎编译的时候附加和重写上下文路径。你也将要注意到height 和width 属性（c处）被包括进来以便给包括在该标签功能中的“遗留的”HTML属性举例。最后styleClass （c处）属性引用在样式表中CSS 类――在这种情况下，即CSS 中的bigimages。

这有显著的优点，你的图像链接通过一个主要资源包引用，这些可以被国际化，并且对其维护可以远离JSP 代码和商业逻辑。在该情况下，一个链接或者指向图像的路径被映射到一个属性文件，这个属性文件是在Strut-config.xml 文件中的一个消息资源项目指定的。当页面被显示的时候，pageKey 属性读出属性文件，处理上下文相关的图像URL，并且把CSS 类的bigimages 经由styleClass 属性附加上去。

◆ 讨论

几乎没有理想的解决办法，对<html:img/>标签来说也是如此。大的高业务量网站时常需要图像有width 和height 属性来加快浏览器的装入过程。增加宽度和高度属性以改善速度是绝对必要的，因为没有它们，浏览器必须把时间花在下载图像上，搞清楚图像的大小然后计算页面余下的控件必须怎样放置。在图像的尺寸被指定后，就为图像创建了一个占位符并且仅仅把它插入到页面，而没有做所有的那些计算。虽然对综合的、低业务量的web 应用程序来说，这不是大问题。但是在拥有百万日点击量的网站，任何性能的增加都可能产生成千上万美金的受益。

为了存储许多静态图像并记住它们，我们已经展示了怎样使用属性文件存储URL以便查找。<html:img/>没有办法干净地把尺寸传达到源代码。以下片断展示属性文件中的一个含有存储的图像URL 路径的项目：

images.anotherOne.logo=/siteRoot/images/logo.gif

你可以在一个填充了size 属性的<html:img/>标签中使用已存储的图像。接下来的片断展示通过srcKey 属性访问该图像（因为没有指定包，默认访问消息资源属性文件）。

<html:img srcKey="images.anotherOne.logo" width="250"  height="125" />

页面因此必须“知道”图像装入的某些东西，并且因此开发者必须知道他使用的所有图像的尺寸。即使URL 已被正确地存储，你必须使用有点深奥的脚本来正确地把尺寸存储在文件中并且读出它。接下来的代码片断展示属性文件的入口：

images.anotherOne.logo=/siteRoot/images/logo.gif

images.anotherOne.logo.width=250

images.anotherOne.logo.height=125

现在，我们将访问URL ，用它的URL 源引用来得到图像的width 和height 属性：

<bean:define name="logoWidth">

    <bean:messagekey="images.anotherOne.logo.width"/>

</bean:define>

<bean:define name="logoHeight">

    <bean:messagekey="images.anotherOne.logo.height"/>

</bean:define>

<html:img srcKey="images.anotherOne.logo"

  width="<%=logoWidth%>"

  height="<%=logoHeight" />

虽然这可能对一些图像起作用，在一个大的站点，里面有成百甚至上千的图像，这就变成一个维护和编码的恶梦，所以这明显是不可能的。在这种情况下最好的解决办法是，对于那些很少改变的图像仅依靠存储在属性文件中的图像路径，并且指定在标签里直接带有size 属性的其他图像，直到Struts 团体决定支持这种方法。
