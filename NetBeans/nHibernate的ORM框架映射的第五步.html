昨天在公司里面终于完成了对UI界面程序的开发。

也完成了对数据库连接、驱动、用户和密码的存储。。

主要的工程结构如下：

而在登录界面的时候，可以动态读取“配置文件”中的信息。

配置文件如下：

更具以上的配置，可以看做是一个JavaBean来存储。

就诞生了ConfigBean，里面主要是包含了几个简单的字段：

/*
* To change this template, choose Tools | Templates
* and open the template in the editor.
*/

package org.jdbc.ui;

/**
*
* @author Banseon
*/
public class ConfigBean {
    private String onlyName;
    private String name;
    private String driver;
    private String url;
    private String user;
    private String pwd;

    public String getName() {
        return name;
    }

    public void setName(String name) {
        this.name = name;
    }

    public String getDriver() {
        return driver;
    }

    public void setDriver(String driver) {
        this.driver = driver;
    }

    public String getUrl() {
        return url;
    }

    public void setUrl(String url) {
        this.url = url;
    }

    public String getUser() {
        return user;
    }

    public void setUser(String user) {
        this.user = user;
    }

    public String getPwd() {
        return pwd;
    }

    public void setPwd(String pwd) {
        this.pwd = pwd;
    }

    public String getOnlyName() {
        return onlyName;
    }

    public void setOnlyName(String onlyName) {
        this.onlyName = onlyName;
    }

}
=============================================

当用户添加一个新的驱动时就被要求添加一个关键字，也就是上述中的onlyName。

通过把关键字和name、pwd的组合生成一个新的字符串。

也就是配置文件中的sql2005.name。这里的sql2005就是关键字了。

但是在取出配置文件中的时候，如何将驱动的名字放入UI层的下拉列表就成了问题的关键。

这里要使用到properties类本身的一个特点，他是继承的Hashtable。

Hashtable是实现了Map接口的，这样我们就可以使用Map.Entry来遍历Hashtable

中的集合元素，HashMap也是如此。

给出我的经典代码，此代码可以用于生成动态的SQL语句。爽啊！！！

/**
     * 查找到所有数据库连接的名字
     * @return
     */
    public List loadSelectItem(){
        List list = new ArrayList();
        Map map = properties;
        for (Object o : map.entrySet() ){
            Map.Entry entry = (Entry) o;
            if(entry.getKey().toString().indexOf("name")!=-1){
                list.add(entry.getValue().toString());
            }
       }
        return list;
    }

==============================================

通过这样的形式就所有的数据库连接的名字得到，并传给下拉列表就得到了。

接着就是通过这个名字得到所有信息咯。

选中某一个驱动后，在其他的文本里面显示相应的信息。

这个原理也是比较简单，这里就不给出代码了。有需要的朋友可以给我留言！
